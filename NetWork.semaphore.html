NetWork.semaphore¶

Semaphores are used to limit simultaneous execution of taks.

All semaphores have a counter value that determines how many paralel tasks can acquire the semaphore. When a task enters a critical section of code, it calls acquire method, if the counter value is greater than zero it gets decremented and the task goes on, if the value is zero the task is put to sleep until one of the tasks that has acquired the semaphore calls release method.

For more info about semaphores see Python documentation page

class NetWork.semaphore.NWSemaphore(workgroup, value=1)[source]¶

The semaphore class used to limit simultaneous execution. When entering critical section call acquire() and when exiting release().

Parameters:
  • workgroup (NetWork.workgroup.Workgroup) – workgroup that will be using this Event
  • value (int) – initial value for the semaphore counter
acquire()[source]¶

Acquire the semaphore and decrement counter value. If the counter is zero sleep until some other task releases the semaphore

release()[source]¶

Release the semaphore, increment the counter value, wake firs task from waiter list.
