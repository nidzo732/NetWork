Event¶

Events are used to enable taks to signal each other that an event has occured. For example, a task that needs some data generated by another task can wait for an event signaled by that task. The waiter process calls the wait method of the event and is put to sleep. When another task call the set method of the same event the first task is waken up and continues it’s work

#Usage examplu
#Prints "Event has been raised" after 5 seconds
import time
from NetWork import Workgroup, Event

def waiter(eventToWait):
    eventToWait.wait()
    print("Event has been raised")

with Workgroup(addresses) as w:
    myEvent=Event(w)
    myTask=w.submit(target=waiter, args=(myEvent,))
    sleep(5)
    myEvent.set()
    sleep(1)    #Dont kill workgroup until the message is sent to worker

For more info about events see Python documentation page

class NetWork.event.NWEvent(workgroup=None)[source]¶

Event class that is used to signal events between tasks. To wait for an event call it’s wait() method and to signal the event use set() method.

Parameters:workgroup (NetWork.workgroup.Workgroup) – workgroup that will be using this Event
set()[source]¶

Set the event, any task that called the wait() method will be waken up

wait()[source]¶

Sleep until some task calls the set() method
